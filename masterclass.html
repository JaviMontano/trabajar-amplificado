Tesis de la clase

Claude no te vuelve soberano por responder bien. Tu soberanía empieza cuando tu contexto queda ordenado en archivos tuyos.

La diferencia entre un chat útil y un Jarvis OS es la persistencia: tu contexto viaja con tus archivos, no muere al cerrar la conversación. Eso significa tu CV, tu rol, tus criterios, tus próximos 1:1 y tus aprendizajes — todo en una carpeta. Si mañana cambias de herramienta, ese contexto sigue siendo tuyo y decides con más claridad.

Contexto propietario

Lo importante vive en una carpeta clara, no escondido en una conversación eterna. Esa carpeta es legible para ti, para Claude y para cualquier otra herramienta agéntica futura.

Soberanía con límites

La IA pregunta, resume, propone y registra. No envía correos, no borra archivos y no modifica documentos críticos sin confirmación humana explícita.

Trabajo real

La clase no termina con teoría. Termina con un CV fuente, un perfil de cargo ordenado y una preparación concreta para una conversación 1:1 con el jefe.

Arquitectura pedagógica

Cómo aprenderás sin quedarte en teoría

La clase usa una secuencia simple: entender el problema, ver la herramienta correcta, montar una carpeta mínima, ejecutar prompts con anexos, revisar supuestos y cerrar con un caso de utilidad inmediata. El lector no necesita volverse técnico; necesita saber qué queda listo y qué debe validar.

Verás

La diferencia entre Chat, Cowork y Code, y por qué una carpeta de archivos simples puede sostener mejor tu trabajo que una conversación larga.

Harás

Crearás tu primer caso operativo: CV fuente, perfil de cargo, responsabilidades reales y preparación de una conversación 1:1.

Claude Code sin mistificar

Claude Code no es solo para programar: es una forma de trabajar con carpetas, archivos y verificación.

Claude Code organiza tu trabajo en carpetas, no solo en chat. Lee tu proyecto, edita archivos, ejecuta comandos y deja trazabilidad. No necesitas programar: aquí lo usamos como una mesa de trabajo. Abres una carpeta, creas archivos, pides verificación y guardas lo que importa.

CLI como taller

Entrar a una carpeta y pedir un resultado concreto: “crea esta estructura”, “lee estos archivos”, “prepara este 1:1”. La terminal no es el objetivo; es el canal.

Docs y notas también cuentan

Una carpeta de markdown se puede tratar como proyecto. El material personal deja de ser texto suelto y se vuelve un sistema de trabajo con memoria local.

Verificación antes de cierre

El hábito no es “pedirle algo a la IA”. El hábito es pedir, revisar diff mental, confirmar supuestos, guardar artefacto y dejar próximo paso.

Regla MetodologIA: el participante no necesita volverse desarrollador para usar lógica de desarrollo: carpeta, archivo, versión, criterio, validación y cierre.

Prompts de facilitación

La IA guía; la persona decide

Prompt · apertura de entorno
Actúa como facilitador de configuración Jarvis OS dentro de Claude Cowork. Primero explícame qué vas a revisar, qué parámetros necesitas y qué anexos conviene adjuntar. Después hazme máximo cinco preguntas para entender mi rol, mi objetivo profesional y la reunión 1:1 que quiero preparar. No escribas ni modifiques archivos críticos sin pedirme confirmación explícita.
Prompt · cierre trazable
Cierra esta sesión como facilitador. Lista los archivos creados, decisiones tomadas, supuestos pendientes, riesgos de privacidad y próximo paso recomendado. Si falta información crítica para mi 1:1 con el jefe, pregúntamela antes de dar el cierre por terminado.
