Caso de práctica · datos sintéticos

Por qué suma

Un P&L permite practicar lectura, comparación, explicación de variaciones y preguntas de control sin inventar un contexto interno de IEB.

Cómo se usa

La clase trabaja con ingresos, costos, utilidad bruta, gastos operativos, periodo, unidad, moneda y versión del archivo.

Límite

Si IEB decide usar datos reales, primero se acuerdan permisos, confidencialidad y criterios de revisión humana.

Alcance ajustable

Decisión · variantes por grupo

Qué decide IEB

Después de la clase 0, IEB puede activar una o varias variantes de profundización: por ejemplo, 2 temas para usuarios de Copilot 365 y otros 2 temas para managers o administrativos con Claude Cowork. Si un grupo necesita nivelación, se acuerda antes de entrar a la profundización.

Criterio práctico

La decisión se toma por licencias disponibles, datos autorizados, perfil de participantes, nivel técnico y valor de aprendizaje del caso financiero.

Límite

No se cierra un alcance productivo desde una promesa general. Se valida fit, restricciones y responsables antes de formalizar.

Soberanía Digital

Concepto rector · método antes que herramienta

Qué significa

IEB conserva reglas, archivos, memoria de trabajo y criterio aunque cambie de Copilot a Claude, Codex u otra herramienta.

Ejemplo financiero

La plantilla para revisar P&L debe vivir como método de IEB: fuentes requeridas, preguntas, límites y formato de salida.

Límite

La herramienta ayuda, pero no reemplaza propiedad intelectual, gobierno de datos ni criterio de decisión.

Prompt

Instrucción clara · pedir mejor

Qué significa

Un prompt es una instrucción para la IA con contexto, tarea, fuente, formato y criterio de revisión. No es una frase mágica.

Ejemplo financiero

Pedir: “compara este P&L contra presupuesto, marca variaciones mayores a X y separa datos, hipótesis y preguntas”.

Límite

Si la fuente no está clara, el prompt debe pedir aclaración antes de producir conclusiones.

Asistente

Configuración reusable · repetir con criterio

Qué significa

Un asistente guarda instrucciones, tono, límites, preguntas de aclaración y formato de salida para repetir una tarea con más consistencia.

Ejemplo financiero

Un asistente de P&L puede pedir periodo, moneda, unidad, versión, fuente y objetivo antes de analizar variaciones.

Límite

El asistente no decide ni aprueba. Prepara material revisable para una persona responsable.

Skill

Receta de trabajo · proceso reutilizable

Qué significa

Una skill empaqueta pasos, entradas, reglas, validaciones y formato de salida para que un proceso se ejecute de forma repetible.

Ejemplo financiero

La skill “revisión mensual de P&L” ordena carga de datos, validación de columnas, análisis de variaciones y preguntas para responsables.

Límite

Si cambian la fuente, el formato o la regla de negocio, la skill debe actualizarse antes de usarse.

Agente

Flujo con límites · pasos y control humano

Qué significa

Un agente divide una tarea en pasos, usa herramientas, prepara entregables y escala a una persona cuando falta evidencia o hay una decisión sensible.

Ejemplo financiero

Un agente puede preparar paquete de revisión: resumen, variaciones P&L, hipótesis, preguntas abiertas y evidencias faltantes.

Límite

El agente debe distinguir datos, hipótesis y decisiones. No debe inventar causas ni cerrar aprobación.

Mini app

Prototipo pequeño · una tarea concreta

Qué significa

Una mini app es un prototipo navegable para resolver una tarea puntual. Sirve para aprender y validar flujo, no para reemplazar sistemas.

Ejemplo financiero

Un comparador de P&L con CSV sintético, selector de periodos, tabla de variaciones y comentario ejecutivo para revisión.

Límite

No entra a producción sin documento de alcance, seguridad, pruebas, responsable funcional y responsable técnico.

Adaptación de clases y contenidos

Estructura estándar · contenidos ajustables

Formar formadores

Prompting de alto rendimiento

Clase 1 de 9 · práctica con finanzas

Qué se enseña

Diseñar prompts de alto rendimiento. Un prompt es una instrucción clara para la IA: rol, contexto, fuente, salida esperada, restricciones, preguntas de aclaración y criterio para revisar la respuesta.

Ejemplo financiero

Construir un prompt para comparar P&L real contra presupuesto y periodo anterior, separando variación de ingresos, costos, utilidad bruta, gastos operativos y supuestos no demostrados.

Límite

Crear skills

Clase 3 de 9 · práctica con finanzas

Convertir una práctica recurrente en una skill: una receta reusable con pasos, entradas, reglas, formato de salida, validaciones y criterios de done.

Entregable

Límite

Crear agentes

Clase 4 de 9 · práctica con finanzas

Definir agentes. Un agente es un flujo que divide un trabajo en pasos, usa herramientas, prepara entregables y pasa a revisión humana cuando falta información o aparece una decisión sensible.

Ejemplo financiero

Diseñar un agente que prepare el paquete de revisión financiera: resumen ejecutivo, variaciones P&L, posibles causas, preguntas abiertas y lista de evidencias necesarias.

Entregable

Mapa de agente financiero con tareas, entradas, salidas, límites, paso a revisión humana y criterios de control.

Límite

El agente no inventa causas: diferencia evidencia, hipótesis y preguntas pendientes.

Ejemplo financiero

Adopción, gobierno y piloto

Clase 8 de 9 · práctica con finanzas

Qué se enseña

Definir un piloto responsable: audiencia, casos permitidos, datos autorizados, rituales, responsables, métricas de uso y criterio de continuidad.

Ejemplo financiero

Diseñar un piloto para revisión asistida de P&L con datos sintéticos o autorizados, responsables definidos y devolución ejecutiva después de varias sesiones.

Entregable

Plan de piloto por frente: alcance, audiencias, permisos, agenda, entregables revisables y decisión de continuidad.

Límite

No se prometen beneficios económicos. Se observa adopción, calidad del entregable, claridad de decisiones y fricciones reales.

Prototipado y mini apps

Clase 9 de 9 · práctica con finanzas

Qué se enseña

Prototipar una mini app sin convertirla en sistema productivo. Una mini app es un prototipo pequeño: entradas, flujo, visualización, salida estructurada, pruebas y backlog.

Ejemplo financiero

Crear un prototipo de comparador de P&L con carga de CSV sintético, selector de periodos, tabla de variaciones y comentario ejecutivo generado para revisión.

Entregable

Mini app o prototipo navegable de análisis P&L, con backlog, límites, pruebas y condiciones para evolucionarlo si IEB lo decide.

Límite

El prototipo no se conecta a datos reales ni entra a producción sin documento de alcance, seguridad, pruebas y responsables aprobados.
