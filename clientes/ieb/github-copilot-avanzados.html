Resultado · prototipo y especificación

Qué debería poder hacer el grupo al cerrar

La profundización usa un caso sintético o autorizado para construir sin tocar información sensible. La meta no es producir software final, sino un prototipo claro y una especificación que permita revisar, priorizar y decidir si vale la pena evolucionarlo.

Usuario, tarea, dato permitido, salida esperada y decisión que no se delega.

Validación · antes de propuesta formal

Condiciones para cerrar alcance con IEB

Licenciamiento (licencias y reglas de uso)

Confirmar acceso a repositorios (espacios donde vive el código), entornos locales, políticas de seguridad y herramientas autorizadas para asistentes de código.

Datos

Trabajar primero con ejemplos sintéticos o material autorizado. Los datos reales requieren reglas explícitas de confidencialidad y revisión.

Discovery (conversación de exploración)

Validar con IEB audiencia, grupos por interés, población objetivo, procesos candidatos, restricciones, herramientas ya habilitadas y criterio de éxito del piloto.

Prudencia

No afirmar dolores internos, responsables, beneficios económicos prometidos o necesidades específicas sin evidencia entregada por IEB.

Curso base antes de profundizar

9 clases comunes · no se repiten aquí

Qué se enseña

La ruta principal instala lenguaje común, prompting (diseño de instrucciones para la IA), asistentes (configuraciones reutilizables), skills (recetas de trabajo), agentes (flujos con pasos), controles, multimodalidad (uso de texto, tablas, imágenes o láminas) y mini apps (aplicaciones pequeñas de prueba). La profundización parte de esa base.

Ejemplo financiero

El grupo ya sabe cómo pedir, revisar y limitar una salida de IA (inteligencia artificial). En estas dos clases aprende a pasar de exploración conversacional a prototipo y de prototipo a especificación revisable.

Entregable

Lista de clases, archivos de práctica, límites de automatización y criterios de revisión humana para la profundización.

Límite

No se reabre todo el curso. Si el grupo necesita nivelación, se agenda aparte antes de entrar a las dos clases.

Vibe Coding (prototipar conversando con IA) responsable

Profundización 1 de 2 · después del curso base

Qué se enseña

Vibe Coding significa prototipar conversando con IA (inteligencia artificial): describir la intención, pedir una primera versión, probar, corregir y mejorar. La parte responsable es no aceptar código sin entenderlo, probarlo y documentar sus límites.

Ejemplo técnico

Crear un prototipo pequeño con datos sintéticos: cargar un archivo, validar columnas, calcular una salida y mostrar un resultado revisable por una persona.

Entregable

Prototipo local o script (secuencia de instrucciones ejecutables) pequeño con instrucciones de ejecución, comentarios útiles, pruebas básicas y criterios de revisión.

Límite

No se ejecuta sobre datos reales ni en ambientes productivos sin revisión técnica, permisos y respaldo.

Desarrollo de Software Amplificado por Especificaciones (construir desde requisitos, pruebas y límites claros)

Profundización 2 de 2 · después del curso base

Qué se enseña

Desarrollo de Software Amplificado por Especificaciones significa usar IA (inteligencia artificial) para construir mejor porque primero se define qué debe pasar: objetivo, usuarios, datos, flujos, casos borde (situaciones límite), pruebas, criterios de aceptación (condiciones para dar por bueno el resultado) y límites.

Ejemplo técnico

Especificar una mini app (aplicación pequeña de prueba) antes de construirla: quién la usa, qué archivo acepta, qué valida, qué pantalla muestra, qué errores controla y qué prueba confirma que funciona.

Entregable

Documento de especificación, backlog (lista priorizada de pendientes), pruebas mínimas, prototipo navegable o local y condiciones para evolucionar si IEB lo decide.

Límite

El prototipo no entra a producción sin documento de alcance, seguridad, pruebas, responsables y mantenimiento.
