Frente avanzado · GitHub Copilot + Claude Code + Codex

Construir activos técnicos con Copilot, Claude Code y Codex

GitHub Copilot, Claude Code y Codex son asistentes de IA para trabajo técnico: ayudan a escribir especificaciones, scripts o automatizaciones pequeñas, revisar código y prototipar. Este frente es distinto de Claude Cowork: aquí la población objetivo son perfiles técnicos o cercanos a automatización. La ruta mantiene Soberanía Digital: el criterio, el repositorio y las reglas siguen siendo de IEB.

Lenguaje común · términos mínimos

Pocas palabras, bien entendidas

Si una persona no puede explicar el trabajo con claridad, la IA tampoco lo ejecuta bien. Estos términos se usan durante toda la ruta.

Soberanía Digital

El método es de IEB

Capacidad de conservar reglas, archivos, memoria y forma de trabajar aunque cambie la herramienta.

Prompt

Instrucción clara

Un prompt dice a la IA el contexto, la tarea, la fuente, el formato y el criterio de revisión.

Asistente

Configuración reusable

Un asistente guarda instrucciones, límites y ejemplos para repetir mejor una tarea.

Skill

Receta de trabajo

Una skill empaqueta pasos, entradas, validaciones y salida esperada para un proceso repetible.

Agente

Flujo con límites

Un agente divide una tarea en pasos, usa herramientas y pasa a revisión humana cuando aparece una decisión sensible.

Mini app

Prototipo pequeño

Una mini app resuelve una tarea concreta. No es una plataforma productiva hasta pasar seguridad, pruebas y responsables.

Un mismo hilo práctico durante toda la ruta

Para que la formación no quede abstracta, las clases usan un hilo financiero: leer un P&L, explicar variaciones, preparar preguntas, producir un resumen ejecutivo y prototipar una mini app de comparación.

Validación · antes de propuesta formal

Condiciones para cerrar alcance con IEB

Licenciamiento

Confirmar acceso a repositorios, entornos locales, políticas de seguridad y herramientas autorizadas para asistentes de código.

Datos

Trabajar primero con ejemplos sintéticos o material autorizado. Los datos reales requieren reglas explícitas de confidencialidad y revisión.

Discovery

Validar con IEB audiencia, grupos por interés, población objetivo, procesos candidatos, restricciones, herramientas ya habilitadas y criterio de éxito del piloto.

Prudencia

No afirmar dolores internos, responsables, beneficios económicos prometidos o necesidades específicas sin evidencia entregada por IEB.

Clase 0 · Introducción y lenguaje común

Clase de entrada · sin costo ni riesgo

Qué se enseña

Alinear qué puede pedir una persona a la IA, qué debe validar siempre y qué no debe delegar. La sesión instala el vocabulario de Soberanía Digital y lo aterriza en GitHub Copilot, Claude Code y Codex con repositorios de prueba y datos sintéticos. Este frente se mantiene separado de Claude Cowork porque trabaja sobre repositorios, scripts y prototipos.

Ejemplo financiero

Se usa un P&L sintético para mostrar cómo pedir análisis, detectar supuestos, marcar datos faltantes y convertir una respuesta en preguntas útiles.

Entregable

Criterio compartido de fit: audiencia candidata, herramienta viable, caso financiero de práctica, restricciones y siguiente frente a validar.

Límite

No se usan datos sensibles ni se fuerza contratación: la clase sirve para confirmar si el enfoque encaja con IEB.

Prompting de alto rendimiento

Clase 1 de 9 · práctica con finanzas

Qué se enseña

Diseñar prompts de alto rendimiento. Un prompt es una instrucción clara para la IA: rol, contexto, fuente, salida esperada, restricciones, preguntas de aclaración y criterio para revisar la respuesta.

Ejemplo financiero

Construir un prompt para comparar P&L real contra presupuesto y periodo anterior, separando variación de ingresos, costos, utilidad bruta, gastos operativos y supuestos no demostrados.

Entregable

Batería inicial de prompts financieros para análisis de P&L, explicación de variaciones y preparación de preguntas para revisión humana.

Límite

Usar datos sintéticos o autorizados; cualquier cifra sin fuente se marca como supuesto o dato faltante.

Crear asistentes

Clase 2 de 9 · práctica con finanzas

Qué se enseña

Pasar de prompts sueltos a un asistente: una configuración reusable con propósito, límites, estilo de respuesta, preguntas de aclaración y formato de entrega.

Ejemplo financiero

Crear un asistente para analizar y comparar estados financieros, especialmente P&L, que pida periodo, unidad, moneda, fuente y criterio antes de emitir conclusiones.

Entregable

Asistente financiero v1 con instrucciones, checklist de insumos y plantilla de salida para análisis comparativo.

Límite

El asistente no aprueba decisiones ni reemplaza revisión contable, financiera o directiva.

Crear skills

Clase 3 de 9 · práctica con finanzas

Qué se enseña

Convertir una práctica recurrente en una skill: una receta reusable con pasos, entradas, reglas, formato de salida, validaciones y criterios de done.

Ejemplo financiero

Diseñar una skill de revisión mensual de P&L: limpiar insumos, detectar variaciones relevantes, pedir explicación de impulsores y generar preguntas para responsables.

Entregable

Skill documentada para revisión mensual de resultados, lista para ejecutarse con el asistente o herramienta aprobada.

Límite

La skill exige fuente, periodo y versión del archivo antes de comparar resultados.

Crear agentes

Clase 4 de 9 · práctica con finanzas

Qué se enseña

Definir agentes. Un agente es un flujo que divide un trabajo en pasos, usa herramientas, prepara entregables y pasa a revisión humana cuando falta información o aparece una decisión sensible.

Ejemplo financiero

Diseñar un agente que prepare el paquete de revisión financiera: resumen ejecutivo, variaciones P&L, posibles causas, preguntas abiertas y lista de evidencias necesarias.

Entregable

Mapa de agente financiero con tareas, entradas, salidas, límites, paso a revisión humana y criterios de control.

Límite

El agente no inventa causas: diferencia evidencia, hipótesis y preguntas pendientes.

Multimodalidad con IA

Clase 5 de 9 · práctica con finanzas

Qué se enseña

Trabajar con multimodalidad: usar texto, tablas, imágenes o presentaciones para convertir análisis en tabla ejecutiva, lámina, guion de presentación, minuta o resumen visual.

Ejemplo financiero

Transformar un análisis de P&L en una lámina ejecutiva con los tres impulsores principales, riesgos de interpretación y preguntas para comité.

Entregable

Borrador de lámina o resumen financiero con narrativa, titulares prudentes y anexos de supuestos.

Límite

La pieza visual no oculta incertidumbre ni convierte correlaciones en causalidad.

Diseño de ecosistemas agénticos de trabajo

Clase 6 de 9 · práctica con finanzas

Qué se enseña

Diseñar un sistema de trabajo donde personas, asistentes, agentes, plantillas, fuentes y reuniones tengan responsabilidades claras.

Ejemplo financiero

Mapear el ecosistema para revisión de resultados: finanzas prepara P&L, líderes explican impulsores, coordinación consolida acuerdos y dirección valida decisiones.

Entregable

Mapa operativo del ecosistema financiero: roles, rituales, fuentes, entregables y puntos de control.

Límite

El ecosistema preserva responsabilidad humana sobre interpretación, comunicación y decisión.

Diseño de arneses para IA

Clase 7 de 9 · práctica con finanzas

Qué se enseña

Construir arneses para IA. Un arnés es un conjunto de barandas: estructura de datos, reglas de validación, rúbricas, formatos de salida, pruebas y criterios de aceptación.

Ejemplo financiero

Definir un arnés para P&L: columnas mínimas, moneda, periodo, unidad, versionado, umbrales de variación y validación de totales.

Entregable

Checklist de arnés financiero con validaciones, red flags, preguntas de control y formato de reporte.

Límite

Si el archivo no cumple el contrato mínimo, la IA debe pedir corrección antes de analizar.

Adopción, gobierno y piloto

Clase 8 de 9 · práctica con finanzas

Qué se enseña

Definir un piloto responsable: audiencia, casos permitidos, datos autorizados, rituales, responsables, métricas de uso y criterio de continuidad.

Ejemplo financiero

Diseñar un piloto para revisión asistida de P&L con datos sintéticos o autorizados, responsables definidos y devolución ejecutiva después de varias sesiones.

Entregable

Plan de piloto por frente: alcance, audiencias, permisos, agenda, entregables revisables y decisión de continuidad.

Límite

No se prometen beneficios económicos. Se observa adopción, calidad del entregable, claridad de decisiones y fricciones reales.

Prototipado y mini apps

Clase 9 de 9 · práctica con finanzas

Qué se enseña

Prototipar una mini app sin convertirla en sistema productivo. Una mini app es un prototipo pequeño: entradas, flujo, visualización, salida estructurada, pruebas y backlog.

Ejemplo financiero

Crear un prototipo de comparador de P&L con carga de CSV sintético, selector de periodos, tabla de variaciones y comentario ejecutivo generado para revisión.

Entregable

Mini app o prototipo navegable de análisis P&L, con backlog, límites, pruebas y condiciones para evolucionarlo si IEB lo decide.

Límite

El prototipo no se conecta a datos reales ni entra a producción sin documento de alcance, seguridad, pruebas y responsables aprobados.
