Resultado · práctica financiera

Qué debería poder hacer el grupo al cerrar

La profundización usa un P&L sintético o autorizado para construir sin tocar información sensible. La meta no es producir software final, sino un activo técnico claro, revisable y gobernable.

Entrada

Contrato de datos

Columnas, tipos, moneda, periodo, versión y reglas mínimas.

Automatización

Script o macro

Limpieza, cálculo de variaciones, alertas y salida estructurada.

Prueba

Casos básicos

Ejemplos esperados para detectar errores antes de compartir.

Entrega

README y backlog

Qué hace, qué no hace, cómo se revisa y qué falta para escalar.

Validación · antes de propuesta formal

Condiciones para cerrar alcance con IEB

Licenciamiento

Confirmar acceso a repositorios, entornos locales, políticas de seguridad y herramientas autorizadas para asistentes de código.

Datos

Trabajar primero con ejemplos sintéticos o material autorizado. Los datos reales requieren reglas explícitas de confidencialidad y revisión.

Discovery

Validar con IEB audiencia, grupos por interés, población objetivo, procesos candidatos, restricciones, herramientas ya habilitadas y criterio de éxito del piloto.

Prudencia

No afirmar dolores internos, responsables, beneficios económicos prometidos o necesidades específicas sin evidencia entregada por IEB.

Curso base antes de profundizar

9 clases comunes · no se repiten aquí

Qué se enseña

La ruta principal instala lenguaje común, prompting de alto rendimiento, asistentes, skills, agentes, controles, multimodalidad y mini apps. La profundización parte de esa base.

Ejemplo financiero

El grupo ya sabe cómo pedir, revisar y limitar una salida de IA. En estos dos temas aprende a convertir eso en macros, scripts y mini apps pequeñas.

Entregable

Lista de temas, archivos de práctica, límites de automatización y criterios de revisión humana para la profundización.

Límite

No se reabre todo el curso. Si el grupo necesita nivelación, se agenda aparte antes de entrar a los dos temas.

Macros y scripts con IA

Profundización 1 de 2 · después del curso base

Qué se enseña

Usar IA para pasar de una tarea repetitiva a una automatización pequeña: macro, script de Python o JavaScript, notebook o función documentada.

Ejemplo financiero

Crear un script que lea un CSV sintético de P&L, valide columnas, calcule variaciones y exporte una tabla revisable.

Entregable

Macro o script con comentarios útiles, instrucciones de ejecución, pruebas básicas y criterios de revisión.

Límite

No se ejecuta sobre datos reales ni en ambientes productivos sin revisión técnica, permisos y respaldo.

Mini app técnica revisable

Profundización 2 de 2 · después del curso base

Qué se enseña

Prototipar una mini app con especificación, interfaz simple, datos sintéticos, pruebas y backlog. Una mini app es un prototipo pequeño, no una plataforma productiva.

Ejemplo financiero

Comparador de P&L con carga de archivo sintético, selector de periodos, tabla de variaciones y comentario ejecutivo para revisión.

Entregable

Prototipo navegable o local, README, backlog, límites y condiciones para evolucionar si IEB lo decide.

Límite

El prototipo no entra a producción sin documento de alcance, seguridad, pruebas, responsables y mantenimiento.
