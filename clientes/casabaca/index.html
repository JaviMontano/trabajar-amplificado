Soberanía Digital para integrar Claude, Gemini y NotebookLM

Un curso práctico para que Casabaca use IA (inteligencia artificial) con Claude, Gemini y NotebookLM. La meta es dejar método, fuentes, reglas y criterio operativo dentro del equipo, no depender de prompts sueltos (instrucciones aisladas para pedirle algo a una IA).

Contexto público · Casabaca

La IA debe servir a una operación con ventas, postventa, repuestos, financiamiento y servicio.

El sitio público de Casabaca presenta una operación Toyota con modelos, talleres y mantenimientos, repuestos y llantas, accesorios, financiamiento, Toyota Go y canales de contacto. Esa diversidad pide un enfoque de IA con memoria, trazabilidad y criterio humano.

Ventas

Modelos y asesoría

Preparar respuestas consistentes para asesores, comparativos de modelos, mensajes comerciales y seguimiento de oportunidades.

Postventa

Talleres y mantenimientos

Consultar políticas, procedimientos, citas, historial permitido, preguntas frecuentes y rutas de escalamiento.

Operación

Repuestos, accesorios y financiamiento

Ordenar información dispersa, evaluar proveedores, documentar decisiones y preparar piezas ejecutivas o comerciales.

Arquitectura pedagógica · 3 herramientas

No es elegir una IA. Es saber qué trabajo hace cada una.

Soberanía Digital significa que Casabaca conserva el criterio: la IA propone, resume, ordena o acelera; la decisión sensible queda bajo revisión humana.

Claude

Razonamiento y diseño de trabajo

Claude se usa para pensar con estructura: redactar asistentes (IA con rol estable), crear skills (recetas reutilizables), comparar opciones, preparar presentaciones ejecutivas y diseñar agentes (IA que ejecuta pasos) con límites claros. En el curso no reemplaza el criterio humano; funciona como mesa de trabajo para acelerar análisis y documentación.

Gemini

Trabajo conectado

Gemini se orienta a tareas donde el entorno Google sea relevante: Drive, Docs, Sheets, Meet, investigación y materiales multimodales (texto, imagen, audio o video). El curso enseña cuándo Gemini conviene por cercanía al flujo de trabajo y cuándo otra herramienta da mejor control.

NotebookLM

Memoria con fuentes visibles

NotebookLM funciona como una memoria consultable: responde desde documentos autorizados y permite ver de dónde sale cada respuesta. El curso enseña a curar fuentes, separar evidencia de inferencia y declarar brechas cuando un documento no contiene la respuesta.

Clase 0 · Intro de riesgo cero

Fit, lenguaje común y datos permitidos

Se valida el fit (encaje real entre necesidad, equipo y formato), qué grupos participarían, qué herramientas ya están disponibles, qué fuentes pueden usarse y qué casos tienen sentido. Salida: mapa inicial de casos para ventas, postventa, repuestos, financiamiento o gestión interna.

Clase 1 · Prompting de alto rendimiento

Pedir bien para revisar mejor

Un prompt (instrucción escrita para una IA) debe decir rol, contexto, fuente, tarea, formato y criterio de revisión. Ejemplo: pedir a Claude o Gemini una comparación de alternativas para un caso comercial o de postventa, y exigir que muestre supuestos y dudas.

Clase 2 · Asistentes para operación

Instrucciones estables para tareas repetidas

Un asistente (IA configurada con rol, instrucciones y límites) apoya procesos repetitivos y simples: preparar respuestas, resumir documentos, comparar opciones o hacer preguntas de control. Se construye un asistente para asesoría comercial, preguntas frecuentes, lectura de documentos o respuestas internas.

Clase 3 · Skills para calidad replicable

Recetas de trabajo

Una skill (receta reutilizable de trabajo) define entradas, pasos, validaciones y salida esperada. Se diseña una skill para presentaciones, reportes o Excels formulados con supuestos, fórmulas, checks (controles de calidad) y conclusiones claras.

Clase 4 · NotebookLM como memoria

Fuentes antes que respuestas

Se arma el blueprint (plano de diseño) de NotebookLM con documentos autorizados: políticas, procedimientos, fichas, minutas y materiales de capacitación. El foco es responder con fuente visible y declarar cuando falta evidencia.

Clase 5 · Agentes y workflows

Delegar pasos, no decisiones sensibles

Un agente (IA que ejecuta pasos hacia una meta) trabaja dentro de un workflow (flujo de trabajo de principio a fin) hasta producir un borrador revisable. Se practica con evaluación de cotizaciones de proveedores: leer ofertas, normalizar criterios, detectar brechas y preparar una recomendación para revisión humana.

Clase 6 · Multimedia e infografías

Comunicar sin ocultar límites

Multimedia (texto, imagen, audio o video) ayuda a convertir hallazgos en piezas gráficas, infografías ejecutivas, one-pagers (documentos de una página) o presentaciones. El criterio central: una pieza visual puede aclarar, pero no debe esconder incertidumbre, supuestos o riesgos.

Clase 7 · Arneses y gobierno

Reglas para trabajar con IA

Un arnés (conjunto de reglas y controles) define datos permitidos, fuentes, formato de salida, checklist (lista de verificación) y criterio de pausa. Se diseña un arnés para que el equipo sepa cuándo avanzar, preguntar o detenerse.

Clase 8 · Mini apps y prototipos

Probar antes de formalizar

Una mini app (aplicación pequeña) permite probar un flujo: por ejemplo, revisar cotizaciones, consultar fuentes o navegar preguntas frecuentes. No se presenta como sistema productivo sin pruebas, seguridad y responsables.

Clase 9 · Proyecto final

Capacidad instalada

El equipo integra un caso completo: asistente (IA con rol estable), skill (receta reutilizable), NotebookLM, agente (IA que ejecuta pasos), pieza ejecutiva y arnés (reglas de control). La salida es un activo reutilizable y un backlog (lista priorizada de mejoras), no una promesa de automatización total.
