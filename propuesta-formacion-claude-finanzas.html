Contexto financiero · 01

La brecha no es usar IA. Es usarla con criterio financiero.

Soberanía Digital no significa hacerlo todo solo. Significa que el equipo entiende qué pide, con qué datos trabaja, cómo revisa la respuesta y cuándo debe detenerse. La herramienta acelera; el método dirige.

Cifras que necesitan juicio

Variaciones, resúmenes, comparativos y explicaciones para comité se repiten cada mes. Claude puede apoyar, pero el equipo conserva el criterio de materialidad y decisión.

Herramientas sin método cansan

Un prompt es una instrucción escrita que guía a la IA. Si la instrucción no tiene contexto, formato y revisión, la respuesta puede sonar bien y aun así no servir.

Conocimiento disperso

Políticas, reportes, actas y documentos de proveedores viven en distintos lugares. NotebookLM puede funcionar como memoria consultable cuando las fuentes están bien curadas.

Casos de uso · 03

Ejemplos financieros que recorren todo el programa

Cada caso aterriza input, prompt, salida esperada, revisión y límite de uso. P&L significa Profit and Loss, o estado de pérdidas y ganancias. KYC significa Know Your Customer, revisión de identidad y riesgo de una contraparte. MCP significa Model Context Protocol, un estándar para conectar herramientas y fuentes. NotebookLM es un cuaderno de conocimiento que responde desde documentos cargados.

P&L

Lectura de cifras

Análisis de variaciones, comparación de periodos, preguntas al negocio y narrativa para comité.

Excel y modelos

Analizar, dominar y generar

Asistentes analizan archivos existentes; skills fijan estructura, checks y outputs; agentes orquestan borradores o auto generación.

KYC + proveedores

Deep research

Investigación de contrapartes y proveedores con fuentes, inferencias separadas, dudas abiertas y brechas visibles.

MCP + NotebookLM

Memoria consultable

Claude consulta una base NotebookLM con políticas, reportes y documentación interna con fuente visible.

Cada caso incluye criterio de uso, datos necesarios y revisión humana antes de decisiones sensibles.

Clase 0 · Intro, lenguaje común y fit

Sin costo ni riesgo · validación de fit

Qué aprende la persona

Alinear lenguaje común sobre IA, límites, datos permitidos y oportunidades reales del equipo financiero. La intro separa tres usos: asistentes para apoyo repetitivo, skills para método replicable y agentes para delegación de secuencias.

Ejemplo financiero

Tomamos una variación simple de P&L y la convertimos en un prompt robusto: qué pedir, qué datos entregar, qué formato esperar y cómo revisar la salida.

Activo y límite

Queda un mapa de casos candidatos y un primer prompt financiero reutilizable. El límite es claro: si faltan datos o sponsor, se declara antes de vender una ruta más grande.

Clase 1 · Crear asistentes

Apoyo repetitivo y simple, con rol claro

Qué aprende la persona

Aprende a configurar un asistente propio para apoyar procesos repetitivos y simples. Un asistente es una IA con rol, instrucciones estables y formato esperado de respuesta. Sirve cuando ya existe un material y el equipo necesita leerlo mejor.

  • Analizar un P&L o un Excel existente sin rehacer el modelo.
  • Comparar periodos, líneas, supuestos o versiones.
  • Resumir hallazgos y preparar preguntas para negocio.
  • Revisar consistencia básica antes de una conversación o comité.

Ejemplo financiero

Construimos un asistente que analiza un P&L o un Excel ya existente: identifica variaciones por línea, compara periodos, resume riesgos visibles y prepara preguntas para negocio.

Activo y límite

Queda un asistente base de análisis P&L/Excel. El límite: el asistente ayuda a leer, comparar y preguntar; no define la estructura del modelo ni ejecuta el proceso completo.

Clase 2 · Crear skills

Estructura replicable y alta calidad

Qué aprende la persona

Aprende a empaquetar un proceso repetido como una skill. Una skill es una receta reutilizable que asegura estructura, pasos, criterios de calidad, formato de salida y errores a evitar. Sirve cuando el equipo quiere que un documento, modelo o procedimiento salga bien de forma consistente.

  • Estructurar documentos financieros con secciones y criterios claros.
  • Dominar Excels y modelos con pestañas, supuestos, checks y salidas estándar.
  • Convertir buenas prácticas de cierre, variaciones o forecast en procedimientos predecibles.
  • Reducir diferencias de calidad entre analistas, ciclos o entregables.

Ejemplo financiero

Diseñamos una skill para dominar Excels y modelos financieros: hoja de supuestos, estructura de pestañas, checks de fórmulas, control de versiones, tabla de variaciones y salida ejecutiva.

Activo y límite

Queda una skill documentada para documentos, modelos o cierre mensual. El límite: la skill estandariza el procedimiento y eleva calidad; no trabaja desatendida ni decide si la cifra es correcta.

Clase 3 · Crear agentes

Delegación desatendida, con puntos de control

Qué aprende la persona

Aprende a definir agentes para delegar secuencias de trabajo. Un agente es una IA orientada a una misión, con pasos, herramientas, criterios de pausa y puntos de control. Sirve cuando la tarea tiene varios pasos y puede avanzar hasta un borrador revisable.

  • Buscar insumos y separar evidencia de inferencia.
  • Aplicar una skill a varios documentos o modelos.
  • Preparar borradores de reporte, matriz o primer modelo.
  • Dejar tareas programables listas para revisión humana.

Ejemplo financiero

Agente para KYC, proveedores o auto generación de borradores: reúne fuentes, aplica checklist, prepara una matriz, arma un primer reporte o bosqueja un modelo financiero para revisión.

Activo y límite

Queda un blueprint de agente con misión, pasos, entradas, salidas y checkpoints. El límite: lo desatendido llega hasta el borrador trazable; las decisiones y validaciones sensibles siguen siendo humanas.

Clase 4 · Multimedia con IA

De cifras a comunicación ejecutiva

Qué aprende la persona

Convertir datos, hallazgos y marcos de análisis en piezas visuales y narrativas ejecutivas: memo, one-pager, presentación breve o guion de comité.

Ejemplo financiero

Transformamos una tabla de variaciones de P&L en una historia ejecutiva: qué cambió, por qué importa, qué revisar y qué decisión se necesita.

Activo y límite

Queda una plantilla de reporte ejecutivo financiero. El límite: una historia clara no debe esconder incertidumbre, supuestos o datos incompletos.

Clase 5 · Ecosistemas agénticos de trabajo

Sistema, no herramientas sueltas

Qué aprende la persona

Orquestar asistentes, skills, agentes y cadencias como un sistema de trabajo financiero. La meta es que cada pieza tenga un trabajo claro: analizar, estandarizar, delegar o revisar.

Ejemplo financiero

Mapa de ecosistema: asistente que analiza el Excel existente, skill que estandariza el modelo, agente que prepara el paquete mensual, NotebookLM de políticas y ritual de revisión semanal.

Activo y límite

Queda un mapa de sistema financiero amplificado con responsabilidades por pieza. El límite: el ecosistema solo sirve si tiene dueños, fuentes, cadencia y puntos de control.

Clase 6 · Diseño de arneses para IA

Contexto, herramientas, salida y evaluación

Qué aprende la persona

Diseñar el arnés que hace que una IA trabaje con contexto, fuentes, formato de salida y evaluación. Un arnés es el conjunto de reglas, datos, herramientas y checklist que contiene el trabajo de asistentes, skills y agentes.

Ejemplo financiero

Arnés para Excel/modelos: datos mínimos, fuentes permitidas, estándar de pestañas, checks de fórmulas, criterios de materialidad, formato de tabla y revisión antes de compartir.

Activo y límite

Queda un arnés de referencia para análisis financiero. El límite: sin datos confiables, el arnés exige detener, preguntar o etiquetar la salida como preliminar.

Clase 7 · Prototipado y mini apps

De idea a prueba navegable

Qué aprende la persona

Prototipar una mini app. Una mini app es una aplicación pequeña, suficiente para probar un flujo, una pantalla o una decisión antes de construir software formal.

Ejemplo financiero

Mini app para comparación P&L, revisión de modelos, scoring de proveedores o tablero de variaciones: carga de datos de prueba, visualización simple, hallazgos y preguntas para revisión.

Activo y límite

Queda un prototipo o blueprint navegable. El límite: el prototipo valida comprensión y experiencia; no se presenta como sistema productivo.

Clase 8 · Adopción de IA

Del entusiasmo individual al equipo

Qué aprende la persona

Planear adopción con rituales, responsables, quick wins, criterios de datos y acompañamiento. La IA se instala como forma de trabajo, no como novedad aislada.

Ejemplo financiero

Ritual semanal de equipo financiero: revisar un caso P&L, un caso KYC o proveedor, actualizar prompts, registrar aprendizajes y decidir qué se escala.

Activo y límite

Queda un plan de adopción por 30 días. El límite: se priorizan hábitos sostenibles sobre entusiasmo inicial o automatizaciones sin dueño.

Clase 9 · Proyecto final

Activo reutilizable del equipo

Qué aprende la persona

Integrar lo aprendido en un flujo financiero completo y presentarlo como activo reutilizable: problema, usuarios, entradas, IA, revisión, salida y decisión.

Ejemplo financiero

Flujo completo de cierre mensual amplificado: asistente que analiza P&L/Excel, skill que asegura estructura del modelo, agente que prepara borradores, reporte ejecutivo y mini app de consulta.

Activo y límite

Queda una demo ejecutiva del flujo, documentación base y backlog de iteración. El límite: el proyecto final es capacidad instalada, no promesa de sustitución del juicio experto.
