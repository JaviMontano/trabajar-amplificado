Por qué importa · 01

La IA ayuda más cuando el criterio financiero dirige

Soberanía Digital no significa hacerlo todo solo. Significa saber qué pedir, qué datos usar, cómo revisar la respuesta y cuándo detenerse. La herramienta acelera; el método ordena.

Cifras con lectura

Variaciones, resúmenes, comparativos y explicaciones para comité se repiten cada mes. Claude puede apoyar, pero el equipo conserva el criterio de materialidad y decisión.

Prompts con método

Un prompt es una instrucción escrita para la IA. Si no tiene contexto, formato y revisión, la respuesta puede sonar bien y aun así no servir.

Fuentes con trazabilidad

Políticas, reportes, actas y documentos de proveedores viven en distintos lugares. NotebookLM puede ser una memoria consultable cuando las fuentes están curadas.

Taller · Carpetas de contexto para trabajo agéntico

USD 500 · 5 a 15 personas · introductorio

Problema que resuelve

En trabajo agéntico, una IA no solo responde: ejecuta pasos dentro de un flujo. Para que eso funcione, el equipo necesita carpetas claras: fuentes autorizadas, versiones, decisiones previas, restricciones, formatos de salida y criterios de revisión. Una buena carpeta reduce ruido y ayuda a la IA a saber qué usar, qué ignorar y cuándo preguntar.

Práctica guiada

  • Armar una carpeta de contexto para analizar un P&L, preparar una presentación o revisar proveedores.
  • Separar subcarpetas simples: fuentes, decisiones, versiones, plantillas, salidas esperadas y dudas abiertas.
  • Separar fuentes autorizadas, supuestos, dudas y datos que no deben entrar en la IA.
  • Convertir pedidos vagos en encargos claros, con salida esperada y criterios de calidad.
  • Definir cuándo la IA puede avanzar, cuándo debe preguntar y cuándo debe detenerse.

Entregable

Queda un Context Pack, paquete de contexto: plantilla de carpeta y checklist para preparar información antes de usar asistentes, skills o agentes. El límite: más contexto no siempre es mejor; el valor está en curar lo relevante, nombrar bien los archivos y proteger lo sensible.

Requisito de herramienta

Este taller se realiza después de la intro sin costo. Para aprovecharlo bien se requiere Claude Pro, plan de USD 20/mes por usuario como referencia oficial. Se recomienda mantenerlo activo durante todo el proceso.

Clase 0 · Intro de fit y lenguaje común

Sin costo ni riesgo · validación de fit

Qué aprende la persona

Alinear lenguaje común sobre IA, límites, datos permitidos y oportunidades reales del equipo financiero. La intro separa tres usos: asistentes para apoyo repetitivo, skills para método replicable y agentes para delegar secuencias de trabajo.

Ejemplo financiero

Tomamos una variación simple de P&L y la convertimos en un prompt robusto: qué pedir, qué datos entregar, qué formato esperar y cómo revisar la salida.

Entregable y límite

Queda un mapa de casos candidatos y un primer prompt financiero reutilizable. El límite es claro: si faltan datos o sponsor, se declara antes de vender una ruta más grande.

Clase 1 · Asistentes para analizar

Apoyo repetitivo y simple, con rol claro

Qué aprende la persona

Aprende a configurar un asistente propio para apoyar procesos repetitivos y simples. Un asistente es una IA con rol, instrucciones estables y formato esperado de respuesta. Sirve cuando ya existe un material y el equipo necesita leerlo mejor.

  • Analizar un P&L o un Excel existente sin rehacer el modelo.
  • Comparar periodos, líneas, supuestos o versiones.
  • Resumir hallazgos y preparar preguntas para negocio.
  • Revisar consistencia básica antes de una conversación o comité.

Ejemplo financiero

Construimos un asistente que analiza un P&L o un Excel ya existente: identifica variaciones por línea, compara periodos, resume riesgos visibles y prepara preguntas para negocio.

Entregable y límite

Queda un asistente base de análisis P&L/Excel. El límite: el asistente ayuda a leer, comparar y preguntar; no define la estructura del modelo ni ejecuta el proceso completo.

Clase 2 · Skills para estandarizar

Presentaciones y Excels formulados con calidad replicable

Qué aprende la persona

Aprende a empaquetar un proceso repetido como una skill. Una skill es una receta reutilizable que define estructura, pasos, criterios de calidad, formato de salida y errores a evitar. Sirve cuando el equipo quiere producir presentaciones, documentos o Excels formulados con consistencia.

  • Crear presentaciones ejecutivas con narrativa, secciones, evidencia y cierre claro.
  • Diseñar Excels formulados con pestañas, supuestos, fórmulas, checks y salidas estándar.
  • Convertir cierre, forecast, variaciones o evaluación de proveedores en procedimientos predecibles.
  • Reducir diferencias de calidad entre analistas, ciclos, modelos y entregables.

Ejemplo financiero

Diseñamos una skill para producir un paquete financiero: Excel formulado con hoja de supuestos, fórmulas trazables, checks de balance y tabla de variaciones; más una presentación ejecutiva con hallazgos, riesgos y decisión solicitada.

Entregable y límite

Queda una skill documentada para presentaciones y Excels formulados. El límite: la skill ordena y eleva calidad; no inventa cifras, no valida supuestos de negocio por sí sola y no trabaja desatendida.

Clase 3 · Agentes para delegar

Delegación de workflows completos, con puntos de control

Qué aprende la persona

Aprende a definir agentes para delegar workflows completos. Un agente es una IA orientada a una misión, con pasos, herramientas, criterios de pausa y puntos de control. Un workflow es una secuencia de trabajo que recibe entradas, ejecuta pasos y entrega una salida revisable.

  • Leer cotizaciones de proveedores y extraer precio, alcance, tiempos, condiciones y riesgos.
  • Normalizar criterios para comparar ofertas que vienen en formatos distintos.
  • Aplicar una skill de evaluación y preparar matriz, brechas y preguntas abiertas.
  • Entregar un borrador de recomendación listo para revisión humana.

Ejemplo financiero

Construimos un agente para evaluar cotizaciones de proveedores: reúne documentos, identifica datos faltantes, compara criterios, marca riesgos, aplica una matriz de scoring y prepara una recomendación preliminar con evidencia visible.

Entregable y límite

Queda un blueprint de agente para evaluación de proveedores, con misión, pasos, entradas, salidas y checkpoints. El límite: lo desatendido llega hasta el borrador trazable; la adjudicación y las validaciones sensibles siguen siendo humanas.

Licencia sugerida

Para esta clase y otros tramos agénticos del bootcamp, conviene evaluar Claude Max 5x, plan de USD 100/mes, en las personas que vayan a prototipar agentes, workflows, MCP o uso intensivo de contexto.

Clase 4 · Comunicación visual con IA

Piezas gráficas, infografías y comunicación ejecutiva

Qué aprende la persona

Convertir datos, hallazgos y marcos de análisis en piezas visuales: piezas gráficas, infografías ejecutivas, one-pagers, documentos de una página, presentaciones visuales y guiones breves para comité.

Ejemplo financiero

Transformamos una tabla de variaciones de P&L o una evaluación de proveedores en una infografía ejecutiva: qué cambió, por qué importa, qué opciones existen, qué falta validar y qué decisión se necesita.

Entregable y límite

Queda una plantilla de pieza gráfica y presentación ejecutiva. El límite: una visual atractiva no debe esconder incertidumbre, supuestos, datos incompletos ni riesgos materiales.

Clase 5 · Memoria financiera con NotebookLM y MCP

NotebookLM como memoria, MCP como orquestación

Qué aprende la persona

Aprende a organizar conocimiento financiero en una base consultable con fuentes claras. MCP significa Model Context Protocol: un estándar para conectar fuentes, sistemas autorizados y consultas de forma estructurada.

Ejemplo financiero

Diseñamos una memoria NotebookLM con políticas, minutas, reportes, contratos y documentación de proveedores. Luego definimos cómo Claude podría consultarla vía MCP para recuperar fuentes, comparar versiones y detectar brechas antes de una decisión.

Entregable y límite

Queda un blueprint de gestión de conocimiento masivo con fuentes, reglas de carga, consultas útiles y criterios de actualización. El límite: si la fuente no existe o no está autorizada, la IA debe declararlo en vez de completar con suposiciones.

Clase 6 · Arneses de control para IA

Controles para workflows financieros

Qué aprende la persona

Diseñar el arnés que hace que una IA trabaje con contexto, permisos, fuentes, pasos, formato de salida y evaluación. Un arnés es el conjunto de reglas, datos, herramientas y checklist que contiene un workflow para que el trabajo no dependa de improvisación.

Ejemplo financiero

Arnés para evaluación de proveedores o creación de modelos: entradas obligatorias, fuentes permitidas, permisos de datos, pasos del workflow, checks de fórmulas, criterios de materialidad, formato de salida y revisión antes de compartir.

Entregable y límite

Queda un arnés de referencia para trabajo financiero y creación de workflows. El límite: sin datos confiables o permiso claro, el arnés exige detener, preguntar o etiquetar la salida como preliminar.

Clase 7 · Dashboards que reemplazan reportes manuales

Reemplazar reportes manuales recurrentes

Qué aprende la persona

Aprende a prototipar dashboards, tableros navegables, como mini apps. Una mini app es una aplicación pequeña: en esta clase se usa para convertir reportes manuales recurrentes en tableros con datos, filtros, lectura ejecutiva y criterios de actualización.

Ejemplo financiero

Construimos un dashboard de variaciones de P&L o proveedores: KPIs, indicadores clave de desempeño, filtros por periodo o unidad, semáforos de materialidad, explicación de variaciones, preguntas abiertas y vista ejecutiva para comité.

Entregable y límite

Queda un prototipo de dashboard y un blueprint para reemplazar un reporte manual específico. El límite: solo se automatiza lo que tenga fuente clara, regla de cálculo, dueño de revisión y control de actualización; no se presenta como inteligencia de negocio, BI, productiva sin pruebas.

Clase 8 · Adopción del equipo

Del entusiasmo individual al equipo

Qué aprende la persona

Planear adopción con rituales, responsables, quick wins, mejoras rápidas, criterios de datos y acompañamiento. La IA se instala como forma de trabajo, no como novedad aislada.

Ejemplo financiero

Ritual semanal de equipo financiero: revisar un caso P&L, un caso KYC o proveedor, actualizar prompts, registrar aprendizajes y decidir qué se escala.

Entregable y límite

Queda un plan de adopción por 30 días. El límite: se priorizan hábitos sostenibles sobre entusiasmo inicial o automatizaciones sin dueño.

Clase 9 · Proyecto integrado

Activo reutilizable del equipo

Qué aprende la persona

Integrar lo aprendido en un workflow financiero completo y presentarlo como activo reutilizable: problema, usuarios, entradas, IA, revisión, salida y decisión.

Ejemplo financiero

Workflow completo de evaluación financiera: skill para Excel formulado y presentación ejecutiva, agente para cotizaciones de proveedores, dashboard, tablero navegable, que reemplaza un reporte manual recurrente, memoria NotebookLM/MCP y arnés de revisión.

Entregable y límite

Queda una demo ejecutiva del workflow, documentación base y backlog de iteración. El límite: el proyecto final es capacidad instalada, no promesa de sustitución del juicio experto.
